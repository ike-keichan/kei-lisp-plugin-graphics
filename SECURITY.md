# Security Policy

## Supported Versions

Only the latest published release of `kei-lisp-plugin-graphics` receives
security updates. Please make sure you are on the newest version before
reporting a vulnerability.

| Version        | Supported |
| -------------- | --------- |
| Latest release | ✅        |
| Older releases | ❌        |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub
issues.**

Instead, use GitHub's private vulnerability reporting:

1. Go to the repository's
   [Security Advisories page](https://github.com/ike-keichan/kei-lisp-plugin-graphics/security/advisories/new).
2. Click **Report a vulnerability** and fill in the form.

Please include as much of the following as you can:

- The version of `kei-lisp-plugin-graphics` (and `kei-lisp`) affected
- The environment (Node.js version, browser, bundler) where it occurs
- A minimal reproduction (Lisp snippet and/or TypeScript code)
- The impact you believe the issue has, and any suggested fix

## What to Expect

This is a hobby project maintained on a **best-effort basis** and is not
recommended for production use (see the note in [README.md](./README.md)).
With that in mind:

- Reports are acknowledged as soon as the maintainer can get to them.
- Once a fix is released, the advisory will be published and you will be
  credited for the discovery (unless you prefer to remain anonymous).

Please give the maintainer a reasonable amount of time to address the issue
before any public disclosure.
